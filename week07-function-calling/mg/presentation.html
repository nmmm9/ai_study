핵심 아이디어

LLM은 텍스트만 생성한다.
Function Calling은 LLM이
실제 함수를 호출할 수 있게 한다.

"미세먼지 어때?" → LLM이 지어내는 게 아니라
fine_dust("경기 안양")을 실행해서
진짜 데이터로 답한다. 

// 사용자
"오늘 미세먼지 어때?"

// LLM이 판단
→ fine_dust("경기 안양")

// 실제 API 결과
→ { pm10: 45, grade: "보통" }

// LLM이 답변 생성
→ "안양 미세먼지 보통이에요"

우리가 쓰는 구조

OpenAI의 tools 파라미터로 26개 도구 스키마를 전달하면, LLM이 알아서 판단

LLM이 판단하는 기준

각 도구의 description을 읽고 질문에 가장 맞는 도구를 선택

"로또 당첨번호?" → description에 "로또 당첨번호 조회" → lotto_results 선택

"안녕" → 어떤 description에도 안 맞음 → 도구 없이 직접 답변

멀티스텝이 가능한 이유

도구 실행 결과를 다시 LLM에 전달 → LLM이 "더 필요하다"고 판단하면 다른 도구를 또 호출

최대 5라운드까지 반복 가능

// 1. 도구 스키마를 LLM에 전달
response = openai.create(
  messages=messages,
  tools=get_all_tools()  // 26개
)

// 2. LLM 응답 확인
if response.tool_calls:
  // 도구 호출 요청이 있으면
  result = execute_tool(
    name, arguments
  )
  messages.append(result)
  // → 다시 1번으로 (반복)

else:
  // 도구 불필요 → 바로 답변
  return response.content

어떻게 동작하나

1

사용자가 자연어로 질문

2

LLM이 26개 도구 중 적절한 도구를 선택 + 파라미터 생성

3

서버가 해당 도구를 실행하고 결과를 LLM에 전달

4

LLM이 결과를 바탕으로 자연스러운 답변 생성

여러 도구가 필요하면 2→3을 반복 (최대 5라운드). 도구 호출 과정이 실시간으로 표시됨.

멀티스텝 예시

"독산점에 피크닉 용품 재고 있어?" → 3번의 도구 호출

STEP 1

매장 검색

daiso_search(
  "독산", "store"
)
→ 독산점 code: "0527"

STEP 2

상품 검색

daiso_search(
  "피크닉", "product"
)
→ 매트 pdNo: "1072428"

STEP 3

재고 확인

daiso_pickup_stock(
  "1072428", "0527"
)
→ 재고: 0개 (품절)

Tool Registry

데코레이터 하나로 도구 등록 → OpenAI tools에 자동 연결

@register_tool(
    name="lotto_results",
    description="로또 당첨번호 조회",
    parameters={ /* JSON Schema */ }
)
async def lotto_results(round=None):
    resp = await client.get(DETAIL_URL, params={...})
    return resp.json()


# 서버 시작 시 자동으로:
tools = get_all_tools()   # → 26개 OpenAI tool schema
result = execute_tool("lotto_results", {})  # → 실행

새 도구 추가 = .py 파일 만들고 @register_tool → 끝. Hermes Agent 패턴 참고.
