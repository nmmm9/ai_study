## 핵심 구현

* 주요 로직 설명:
  전체 구조는 크게 **Plan → Execute → Output** 흐름으로 구성했다.

1. **Plan 단계**
   사용자의 질문을 기반으로 AI가 전체 실행 계획을 JSON 형태로 먼저 만든다.
   (예: 여행 도시, 기간, 수행할 작업 목록 등)

2. **Execute 단계 (ReAct 루프)**
   while 반복문을 통해 AI가 스스로 판단하면서 필요한 작업을 하나씩 실행한다.
   각 단계는 다음 흐름을 따른다:

   * Thought: 다음에 무엇을 할지 판단
   * Action: 필요한 함수 호출
   * Observation: 결과 확인 후 다음 단계 결정

   이 과정을 반복하다가 AI가 충분히 정보를 모았다고 판단하면 종료된다.

3. **Output 단계**
   수집된 데이터를 기반으로 HTML 형태의 여행 계획서를 생성한다.

특히 이번 주차에서 가장 핵심은
**단일 호출이 아니라 반복 루프를 통해 AI가 스스로 실행을 이어간다는 점**이었다.

## WHY (의사결정 기록)

1. **Q**: 왜 이 방식을 선택했는가?
   **A**:
   ReAct만 사용하면 실행 과정은 유연하지만 방향이 흔들릴 수 있고,
   Plan-and-Execute만 사용하면 처음 계획에 너무 의존하게 된다.
   그래서 두 방식을 같이 사용해서
   **전체 구조는 계획적으로 잡고, 세부 실행은 유연하게 처리하도록 했다.**

2. **Q**: 다르게 구현한다면 어떻게 했을까?
   **A**:
   ReAct만 단독으로 사용해서 더 단순하게 구현할 수도 있었을 것 같다.
   또는 LangGraph 같은 구조를 사용해서
   반복 흐름을 while 문이 아니라 그래프 형태로 표현하면
   더 구조적으로 관리하기 좋을 것 같다.
