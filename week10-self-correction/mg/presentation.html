9주차의 한계

Writer 답변을
아무도 검수하지 않았다.

① 검증 없는 노출

Writer 가 도구 결과를 누락하거나 환각을 넣어도 그대로 사용자에게 출력. 사용자가 직접 검증해야 했다.

② 들쭉날쭉한 품질

같은 질문에도 답변 품질이 운에 의존. 좋을 때 9점, 나쁠 때 4점이지만 차이를 알 방법이 없었다.

③ DAG (한 방향 그래프)

9주차 그래프는 한 방향 흐름. Supervisor → Domain → Writer → END. 되돌아가는 루프가 없었다.

④ LangGraph 의 진짜 강점 미사용

LangGraph 는 cycle 을 자연스럽게 표현하는 게 핵심. 9주차는 일반 함수 호출 체인으로도 가능한 수준이었다.

Graph Theory

DAG  vs  Cycle

DAG · Directed Acyclic Graph

한 방향 흐름

  • 한 번 거친 노드는 다시 방문하지 않는다.
  • 실행 시간이 결정적 (예측 가능).
  • 일반 함수 호출 체인과 본질적으로 같다.
  • 오류가 발생해도 그대로 종착점까지 흐른다.
  • 9주차 Multi-Agent 가 여기에 해당.

Cycle · Conditional Loop

되돌아갈 수 있다

  • 조건이 맞으면 이전 노드로 다시 흐른다.
  • 실행 시간이 가변적 (조건 만족까지 반복).
  • 재귀 한계가 무한 루프를 막는다.
  • 자기 검증 + 수정 이 한 그래프에 표현된다.
  • 10주차 Self-Correction 의 핵심.

LangGraph 의 진짜 가치는 cycle 표현에 있다. DAG 는 asyncio.gather 로도 충분하지만, "조건부로 되돌아가기" 는 프레임워크 없이 직접 짜기 까다롭다.

Critic 의 시선

4가지 평가 기준

1

도구 결과 활용도 — 가장 중요

수집된 정보를 모두 반영했는가? 누락된 데이터를 답변에서 무시했는가?

2

사실 정확성

도구가 주지 않은 정보를 만들어내지 않았는가? 환각 / 잘못된 추론은?

3

답변 구조

사용자 질문에 직접 답하는가? Markdown 활용도? 가독성?

4

누락

사용자가 물어본 항목 중 빠뜨린 게 있는가? (특히 복합 질문)

10 · 완벽 7~9 · 통과 4~6 · 재작성 1~3 · 환각

왜 정형화 채점인가

모호함은
피드백 루프를 만들 수 없다.

자유 형식 평가의 문제

"답변이 괜찮은데 조금 아쉬워요" 같은 평가는 다음 단계의 분기 에 쓸 수 없다. 통과인지 재작성인지 기준이 모호하고, 무엇을 고쳐야 할지도 불분명.

정형화된 평가의 힘

1~10 점 + issues 배열 + suggestions 배열로 강제하면, "점수 ≥ 7 이면 통과" 라는 단순 조건으로 cycle 을 종결시킬 수 있고, 재작성 시 무엇을 어떻게 고쳐야 하는지 다음 노드에 그대로 전달된다.

LLM 이 자유롭게 평가하면 사람이 읽기엔 편하지만, 그래프의 다음 노드가 읽기엔 불편 하다. Self-Correction 은 결국 LLM 들끼리 데이터를 주고받는 시스템이라, 평가는 구조화 되어야 한다.

Flow

한 질문에서 일어나는 일

1

Supervisor — 도메인 라우팅 결정 (예: lifestyle)

2

Lifestyle Agent — fine_dust 도구 호출, 결과 수집

3

Writer — 초안 작성 (iteration 1)

4

Critic — 점수 매김. 9/10 통과 또는 5/10 재작성

↻

[루프] Writer 재작성 (iteration 2) → Critic 다시 점수 → 통과 시 END, 실패 시 한도 도달까지

5

END — 최종 답변 + 점수 + iterations 사용자에게 전달

LIMIT

왜 한도를 둬야 하는가

완벽주의는
무한 루프를 부른다.

Critic 이 항상 부족하다고 말하면?

이론적으로 Critic 은 매번 새로운 흠 을 찾아낼 수 있다. 한도 없이 돌리면 통과 점수에 도달 못 하고 무한히 재작성한다.

비용 / 응답 시간의 trade-off

재작성 1회마다 Writer + Critic = 추가 LLM 2회 가 호출된다. 사용자 입장에서 응답 지연 + 토큰 비용이 비례 증가.

완벽보다 실용

현실적으로 7점 이상이면 사용자가 "괜찮은 답변" 으로 받아들인다. 9점·10점 추구는 ROI 낮음.

10주차 채택안

PASS_THRESHOLD = 7 · MAX_REVISIONS = 2. 대부분의 질문은 첫 시도에 통과. 실패해도 최대 2회 재작성에서 멈춰 사용자에게 답변.

현실적 평균

첫 답변이 보통 7점 이상 → Critic 1회만 추가. 평균 비용 +13~20%.

최적화 옵션

Critic 모델을 nano로 → 비용 거의 0. 단 채점 정확도 5~10% 떨어짐 (trade-off).

Research Context

이 패턴은 어디서 왔나

Self-Refine (Madaan et al., 2023)

한 LLM 이 자기 출력을 평가하고 다시 쓰는 패턴. 같은 모델 이 두 역할을 하는 게 특징.

Reflexion (Shinn et al., 2023)

실패 사례를 장기 메모리 에 누적해 다음 시도에 활용. 단순 재작성보다 학습 효과.

Constitutional AI (Anthropic, 2022)

모델이 스스로의 출력을 원칙 에 비춰 비평·수정. RLHF 의 보완으로 사용.

10주차 구현은

Self-Refine 의 분리 변형. Writer 와 Critic 을 다른 system prompt 로 운영해 역할 충돌 을 줄였다.
