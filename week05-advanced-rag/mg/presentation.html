PART 2 — HyDE

HyDE란?

Hypothetical Document Embeddings

질문을 그대로 임베딩하지 않고,
LLM이 가상의 답변을 먼저 생성한 뒤
그 가상 답변을 임베딩하여 검색한다.

Gao et al., 2022 — "Precise Zero-Shot Dense Retrieval without Relevance Labels"

PART 3 — Reranking

Reranking이란?

벡터 검색 결과를 더 넓게 가져온 뒤,
LLM이 각 청크의 실제 관련성을 평가하여
순서를 재정렬하는 기법.

왜 필요한가?

  • 코사인 유사도는 "의미적 거리"만 측정 — 실제 관련성과 다를 수 있음
  • LLM은 질문과 문서를 읽고 이해하여 관련성을 판단
  • Top-5 대신 Top-20을 검색하고, 그 중 진짜 좋은 5개를 선별

PART 3 — Reranking

Reranker의 한계

초기 검색에 없으면 복구 불가능

Reranker는 이미 검색된 결과만 재정렬한다.
초기 벡터 검색에서 아예 포함되지 않은 좋은 청크는 살릴 수 없다.

보완 방법:

넓은 초기 검색

Top-K × 4로 후보군을 넓힘
(5개 → 20개)

Hybrid Search

벡터 + 키워드(BM25) 검색을 결합
(이번 데모에서는 미구현)

HyDE 결합

HyDE로 검색 품질을 먼저 높인 뒤
Reranking으로 정밀도 보강

PART 4 — 추가 기법

Multi-Query RAG

질문을 3가지 관점으로 변형 → 각각 검색 → 합산

원래 질문

"Transformer의 핵심 메커니즘은?"

LLM이 생성한 변형

1. "Transformer 아키텍처에서 가장 중요한 구성 요소는?"

2. "Self-Attention이 Transformer에서 하는 역할은?"

3. "Transformer가 RNN보다 뛰어난 이유는?"

4개 질문 × 각각 검색 → 중복 제거 → 다양한 관점의 소스 확보 → 더 풍부한 답변

PART 4 — 추가 기법

CRAG — Corrective RAG

검색된 문서가 부족하면? → 쿼리를 교정하고 재검색

Search
→
Evaluate
문서 품질 판단
→
Refine
쿼리 수정
→
Re-search
→
Generate

CORRECT

문서가 충분 → 바로 생성

AMBIGUOUS

부분적 → 쿼리 수정 후 재검색

INCORRECT

관련 없음 → 완전히 다른 쿼리로 재검색

Self-RAG는 답변을 평가, CRAG는 검색된 문서를 평가 — 평가 대상이 다르다

04

PART 4 — 추가 기법

Adaptive RAG

질문 복잡도에 따라 파이프라인을 자동 선택

복잡도예시선택되는 파이프라인
SIMPLE "Transformer란 무엇인가?" Basic RAG (빠르고 저렴)
MODERATE "RNN과 Transformer의 차이는?" Rerank RAG (정밀도 향상)
COMPLEX "Transformer를 적용할 때 고려사항은?" Advanced RAG (HyDE + Rerank)

모든 질문에 최고급 파이프라인을 쓸 필요 없다 → 비용/시간 최적화

PART 5 — 구현

Compare 동시 실행

# POST /api/compare — 아무 2개 모드 비교
async def compare(req: CompareRequest):
    runner_a = _RUNNERS[req.mode_a]  # e.g. "self_rag"
    runner_b = _RUNNERS[req.mode_b]  # e.g. "crag"
    a, b = await asyncio.gather(
        runner_a(req.question, req.collection_name),
        runner_b(req.question, req.collection_name),
    )
    return CompareResponse(basic=a, advanced=b)

9가지 모드 중 아무 2개를 선택하여 동시 비교 — 36가지 조합 가능

PART 5 — 구현

핵심 코드: hyde_service.py

HYDE_SYSTEM = (
    "다음 질문에 대한 가상의 답변을 작성하세요. "
    "실제 문서에 있을 법한 정보성 텍스트를 2-3문장으로 작성하세요. "
    "확실하지 않은 내용은 포함하지 마세요."
)

async def generate_hypothetical(question, model="gpt-4o-mini"):
    return await ask_short(
        system_prompt=HYDE_SYSTEM,
        user_prompt=question,
        model=model,
        temperature=0.7,   # 약간의 창의성
        max_tokens=200,     # 2-3문장이면 충분
    )

temperature 0.7로 다양한 표현의 가상 답변 생성 → max_tokens 200으로 비용 제한
