# 6주차: Streamlit UI (1차 시연) - ys

## 기술 스택

| 항목                 | 선택                       | 대안                         | 선택 이유                                                                              |
| ------------------ | ------------------------ | -------------------------- | ---------------------------------------------------------------------------------- |
| UI Framework       | React + Vite             | Streamlit / Flask Template | 이미 React 기반 프론트엔드가 구성되어 있고, 채팅 화면·참고 자료 패널·수집 패널처럼 컴포넌트 분리가 필요하므로 React + Vite를 유지 |
| Backend 연동         | FastAPI API 연동           | Streamlit 내부 Python 함수 호출  | 기존 RAG 파이프라인이 FastAPI 엔드포인트로 구성되어 있어 프론트엔드와 백엔드를 분리한 구조를 유지                        |
| Streaming          | SSE Streaming            | 일반 JSON 응답 / WebSocket     | 답변이 생성되는 과정을 실시간으로 보여줄 수 있고, 단방향 답변 스트리밍에는 SSE가 적합하므로 선택                           |
| Session Management | React State + Storage    | Streamlit Session State    | 사용자의 질문, 답변, 참고 자료, 검색 설정을 화면에서 유지해야 하므로 React 상태 관리와 브라우저 저장소 방식을 사용              |
| Source UI          | Citation 기반 Source Panel | 단순 텍스트 출력                  | RAG 답변의 근거를 사용자가 확인할 수 있도록 [자료 1], [자료 2] 형식의 참고 자료 패널을 사용                         |
| Status Display     | 단계별 진행 상태 표시             | 로딩 스피너만 사용                 | 자료 수집, 검색, 검증, 답변 생성 과정을 구분해 보여주면 1차 시연에서 시스템 흐름을 설명하기 쉬우므로 선택            |

## 핵심 구현

* 주요 로직 설명:

  * 사용자가 질문을 입력하면 React 프론트엔드에서 질문, 종목명, 종목코드, 산업, 자동 수집 여부, 검색 모드를 함께 FastAPI의 `/chat` 엔드포인트로 전달
  * 백엔드는 기존 RAG 파이프라인을 실행하고, SSE 방식으로 처리 결과를 순차적으로 프론트엔드에 전송
  * 프론트엔드는 `collect`, `search_mode`, `sources`, `validation`, `token` 이벤트를 받아 현재 처리 상태를 화면에 표시
  * 사용자의 질문과 AI 답변은 대화 목록에 저장하여 이전 대화가 화면에 유지되도록 구성
  * 검색된 참고 자료는 Source Panel에 [자료 1], [자료 2] 형식으로 표시하고, 답변 본문의 Citation 번호와 연결되도록 구성
  * 검색 모드가 `vector`, `hybrid`, `hybrid_rerank` 중 무엇인지 화면에 표시하여 시연 중 검색 방식 차이를 설명할 수 있도록 구성
  * 자동 수집 여부, 참고 자료 개수, 검증 결과를 함께 보여주어 RAG 답변이 어떤 과정을 거쳐 생성되었는지 확인할 수 있도록 개선
  * 대화 초기화 기능을 추가하여 현재 세션의 질문, 답변, 참고 자료 목록을 사용자가 직접 삭제할 수 있도록 구성

* 코드 실행 방법:

  * 백엔드

    * cd backend
    * .\venv\Scripts\Activate.ps1
    * uvicorn main:app --reload --port 8000

  * 프론트엔드

    * cd ../frontend
    * npm run dev

  * 브라우저

    * http://localhost:5173

## WHY (의사결정 기록)

1. **Q**: 왜 이 방식을 선택했는가?
   **A**: 이번 단계의 주제는 웹 기반 RAG 서비스 1차 시연과 세션 관리였기 때문에, 사용자가 질문을 입력하고 답변과 참고 자료를 확인할 수 있는 UI가 필요했다. Streamlit은 빠른 시연에는 적합하지만, 현재 프로젝트는 이미 React + FastAPI 구조로 구현되어 있고 검색 모드 선택, SSE 스트리밍, 참고 자료 패널, 수집 패널처럼 복잡한 UI 구성이 필요했다. 따라서 기존 구조를 유지하면서 React 상태 관리와 브라우저 저장소를 활용해 대화 기록과 검색 결과를 유지하는 방식이 더 적합하다고 판단했다.

2. **Q**: 다르게 구현한다면 어떻게 했을까?
   **A**: Streamlit 기반의 별도 시연용 UI를 만들어 구현했을 것이다. 기존 React 프론트엔드 대신 'app.py'에서 질문 입력창, 검색 모드 선택, 자동 수집 체크박스, 답변 출력 영역, 참고 자료 영역을 구성하고, Python 코드에서 FastAPI의 '/chat' 엔드포인트를 호출해 결과를 받아오는 방식으로 구현할 수 있다. 대화 기록은 Streamlit의 'st.session_state'에 저장하고, 답변 생성 과정은 상태 메시지로 표시하며, 검색된 참고 자료는 'st.expander'를 사용해 [자료 1], [자료 2] 형식으로 보여주는 구조로 만들 수 있다.
